# DMSE 導彈四階分級 — 架構與逐階實作規格

依設計文件「DMSE 導彈戰鬥內容」，導彈依尺寸分四階，各有對應的發射平台與儲藏架。
本文件記錄**已完成的共用架構腳手架**，以及**四階各自待實作的內容清單**，供逐階填入。

> 決策（與作者確認）：先補齊共用架構再逐階填入；現有 cruise 作為**中型階**的基礎範本；
> 「傾斜 vs 垂直發射」為**純建造限制＋視覺**，不影響彈道數值。

---

## 一、已完成的共用架構（本次）

### 1. 分類列舉與彈體欄位 — `Scorer/Assembly/MissileDefs.cs`
- `MissileSizeClass { Loitering, Light, Medium, Ballistic }`
- `MissileLaunchMode { Tilt, Vertical, Both }`（Both 僅用於彈體：表示可被任一型平台使用）
- `MissileCustomizeLocation { None, Item, StorageBuilding, Launcher }`
- `MissileBodyDef` 新增欄位：`sizeClass`、`launchMode`、`customizeLocation`。

### 2. 平台擴充與相容判定 — `Scorer/Assembly/MissileLauncherExtension.cs`
- `MissileLauncherExtension : DefModExtension`：`sizeClass`、`launchMode`（Tilt/Vertical）、
  `capacity`、`isStorage`。掛在發射架／儲存架的 ThingDef 上。
- `MissileTaxonomy`：`LaunchModeCompatible(...)`、`PlatformAccepts(ext/def, body)`
  —「分級需相符 + 發射方式相容」的共用判定，供裝填驗證／ITab／WorkGiver 使用。

### 3. 發射方式的建造限制（純 XML，無自訂碼）— `ThingDefs_Buildings/DMSE_LauncherBases.xml`
- 抽象母 Def `DMSE_TiltLauncherBase`：`rotatable=true`，可建於重力船。
- 抽象母 Def `DMSE_VerticalLauncherBase`：`rotatable=false`（單一朝向）
  ＋原版 `PlaceWorker_InvalidOverSubstructure`（阻止建於重力船 substructure 地板）。
- 各階發射架以 `ParentName` 繼承其一即可獲得正確的發射方式限制。

### 4. 現有資產的分類標記（中型範本）
- `DMSE_MissileBody_Cruise` → `sizeClass=Medium`、`launchMode=Both`、`customizeLocation=Item`。
- `DMSE_Building_Scorer`（Scorer MLS）→ 掛 `MissileLauncherExtension`（Medium / Tilt / cap 1）。

### 既有可重用元件
- `CompDurabilityDecay`（`CompProperties_DurabilityDecay`）：未受庇護彈藥隨時間衰減耐久。
  `shelterBuildings` 指向對應儲存架即可實現「需存放於儲存架」。
- `CompMissileConfig` / `ITab_MissileAssembly` / `WorkGiver_AssembleMissile` / `MissileConfig`：
  物品型態自定義流程（中型可沿用；之後若改為「在儲存建築上自定義」再擴充）。
- `MissileRack`（2x4 racks，固定僅收 cruise，含 `CompMissileRackRenderer`）：儲存架範本。
- `CompScorer` / `ScorerProjectile` / `WorldObject_IncomingMissile` / `MissileIncoming`：發射→世界旅行→落點結算。

## 二、四階規格對照（逐階待實作）

| 項目 | 巡飛彈 Loitering | 輕型導彈 Light | 中型導彈 Medium | 彈道導彈 Ballistic |
|---|---|---|---|---|
| 發射方式 | 僅傾斜 | 傾斜＋垂直 | 傾斜＋垂直 | 僅垂直 |
| 重力船 | 傾斜可 | 傾斜可/垂直不可 | 傾斜可/垂直不可 | **不可建造與使用** |

### 各階待補的具體 Def / 程式

**巡飛彈 Loitering**
- `MissileBodyDef`（sizeClass=Loitering, launchMode=Tilt, customizeLocation=None）。
- 物品彈藥 ThingDef：`tickerType=Rare` ＋ `CompProperties_DurabilityDecay`
  （`shelterBuildings` = 巡飛彈儲存架）。
- 傾斜發射架 1x3：`ParentName=DMSE_TiltLauncherBase` ＋ `MissileLauncherExtension`(Loitering/Tilt)
  ＋ CompProperties_Refuelable(filter=巡飛彈)。
- 儲存架 1x2：仿 `MissileRack`（fixedStorageSettings 限該彈、CompMissileRackRenderer）。
- 前期 ResearchProjectDef。

**輕型導彈 Light**
- `MissileBodyDef`（Light, Both, None）。
- 物品彈藥：**不需** DurabilityDecay（無特殊儲存）。
- RecipeDef 於 MachinePrinter 製彈體。
- 發射架（四發彈容）：傾斜 `DMSE_TiltLauncherBase` 1x3 與垂發 `DMSE_VerticalLauncherBase` 1x1
  兩個變體；`MissileLauncherExtension`(Light, capacity=4)；Refuelable fuelCapacity=4。

**中型導彈 Medium**（以現有 cruise 為範本擴充）
- 沿用 `DMSE_MissileBody_Cruise`，逐步將 `customizeLocation` 由 Item 改為 StorageBuilding：
  新增「中型導彈儲存/裝配建築」，把 `ITab_MissileAssembly` 流程改掛於該建築（或追加建築版 ITab）。
- 物品彈藥：`CompProperties_DurabilityDecay`(shelter=中型儲存建築)。
- RecipeDef 於 RotaryForger 製彈體。
- 發射架：傾斜 1x4（現有 Scorer 1x5 可調為 1x4）與垂發 1x1 變體。

**彈道導彈 Ballistic**
- `MissileBodyDef`（Ballistic, Vertical, Launcher）；**無物品型態**。
- 發射架 3x3：`ParentName=DMSE_VerticalLauncherBase`（已含 substructure 限制）
  ＋ `MissileLauncherExtension`(Ballistic/Vertical/cap1)。
- 需新增「在發射架上直接建造/自定義彈體」的 Comp 與 ITab（消耗資源、於架上渲染未發射彈體、發射）。
- 重力船禁用：垂直母 Def 已阻止建於 substructure；若需額外阻止「隨重力船起飛後使用」，
  另加運作期檢查（參考 `CompPowerPlantGravcore` 的 onSubstructure 檢查模式）。

## 三、相容性與注意事項
- 平台是否接受某彈體一律走 `MissileTaxonomy.PlatformAccepts`，避免各處重複判斷。
- 垂直限制只在「建造期」由 PlaceWorker 把關；彈道導彈「不可在重力船上使用」如需運作期把關，
  以 `terrainGrid.FoundationAt(pos)?.IsSubstructure` 檢查（同原版 gravcore 作法）。
- `CompDurabilityDecay` 仰賴 parent 會 tick：物品彈藥 ThingDef 必須設 `tickerType=Rare`。
- 新增 .cs 檔需登記進 `DMSE.csproj`（本次已登記 `MissileLauncherExtension.cs`）。
